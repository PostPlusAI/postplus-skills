# Production Handoff

Use this reference only after the user approves the educational-ad direction.

## Rule

Immediately hand the locked plan and script to the appropriate production skill. Do not output a separate production brief or ask a second generation confirmation.

## Route Selection

### Image First

Use `$image-batch-runner` before video when any recurring authority persona, user, product, location, comparison system, proof plate, interface, animation character, first-frame composition, or before/after state needs continuity assets.

Create caption-free source imagery. Keep final claims, labels, package text, interface text, prices, reviews, guarantees, legal copy, and subtitles for approved post-production treatment.

After still approval, pass the assets and locked script to `$video-batch-runner`.

### Direct Video

Use `$video-batch-runner` directly when persona, offer, scenes, proof imagery, audio, and reference duties already exist and are validated, or when shots are independent and simple.

## Handoff Payload

Pass:

- approved `Educational Grammar Plan` and shot table
- claim boundaries and evidence ledger
- proof status, evidence owners, and causal-closure results
- offer facts, type, capability, entry, and interaction
- authority basis and approved copy
- continuity locks
- persona, product, UI, location, audio, and reference assets
- duration, aspect ratio, model, resolution, and audio preferences when supplied
- final text elements to add in post

Tell the runner that creative direction is locked unless it finds a factual, claim, asset, route, duration, or generation-executability issue.

## Reference Duties

- Bind supplied persona, real product, and approved audio references to identity, appearance, and voice continuity.
- Treat benchmark ads as inspiration-only for rhythm, framing, evidence linkage, and argument structure.
- Exclude failed, unreviewed, conflicting, or user-rejected references.
- Never bind a benchmark creator, set, wardrobe, character, package, proprietary interface, or voice.

## Runner Boundary

Let downstream skills handle uploads, missing asset checks, model selection, request schemas, duration and resolution constraints, manifests, polling, and downloads.

Do not duplicate those responsibilities inside `educational-ad`.
