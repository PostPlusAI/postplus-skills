# Editing Workflow

## Goal

Turn script meaning, A-roll delivery, B-roll proof, and reference patterns into a usable short-form post-edit plan.

This skill is primarily a decision system.

It answers:

- what should be shown now
- why this visual belongs under this line
- how long it should stay on screen
- what editing pattern best supports the point

## Order Of Operations

Follow this order unless there is a strong reason not to.

1. define the edit thesis
2. classify the video type
3. build the beat map
4. score A-roll keep moments
5. score B-roll proof value
6. extract reference patterns
7. write beat-by-beat decisions
8. package risks and fallback options

## 1. Define The Edit Thesis

Reduce the piece to one central contrast or claim.

Good edit theses:

- `one small email task turns into a side mission`
- `more tools creates more exits from the task`
- `the useful setup is the one that keeps you in the workflow`

Bad edit theses:

- `this video is about AI`
- `this is a productivity tool`

If the edit thesis is vague, the timeline will be vague.

## 2. Classify The Video Type

Choose the dominant mode.

Common modes:

- tutorial-led
- comparison-led
- viewpoint-led
- proof-led
- demo-led

This affects edit behavior.

Examples:

- tutorial-led: more literal proof, more step visibility, less abstraction
- viewpoint-led: fewer proof moments, stronger restraint, more face time
- comparison-led: clearer before/after visual pairing
- demo-led: longer screen visibility, less rhetorical cutaway

## 3. Build The Beat Map

A beat is not just a sentence. It is a unit of edit intent.

Split when there is a change in:

- idea
- emotional weight
- proof need
- pace
- visual logic

Typical beat roles:

- hook
- setup
- pain
- contrast
- solution reveal
- support proof
- payoff
- CTA

## 4. Score A-roll Keep Moments

Do not cut away from A-roll by default.

Keep A-roll visible when one of these is true:

- the hook depends on face or attitude
- the speaker lands an opinion or realization
- there is useful micro-expression
- there is a pause that creates emphasis
- the line is abstract and needs trust, not proof

Cut away sooner when:

- the line is procedural
- the proof matters more than the speaker's face
- the A-roll is visually flat
- the beat needs pace acceleration

## 5. Score B-roll Proof Value

For each B-roll asset, capture:

- literal content
- proof claim
- best beat roles
- strength
- reusability
- ideal duration range
- repetition risk

Strength labels:

- `primary`
- `supporting`
- `bridge`
- `filler`

Primary proof means the claim becomes weaker without it.

Supporting proof adds credibility but is not structurally required.

## 6. Extract Reference Patterns

Do not imitate surface style blindly.

Extract only reusable patterns such as:

- hook starts on face then cuts to proof by second 1.2
- comparison montage uses 3 quick friction shots
- solution reveal returns to face before showing product
- subtitles thin out during denser B-roll

Avoid over-learning:

- exact transitions
- exact fonts
- arbitrary speed ramps

Reference videos should teach edit logic, not just decoration.

## 7. Write Beat-By-Beat Decisions

Each beat decision should include:

- beat id
- timing
- spoken line
- edit intention
- A-roll treatment
- B-roll assignment
- subtitle note
- pace note
- transition note
- fallback if the preferred asset is weak

The key question:

- why this cut here

If the answer is only `to make it dynamic`, the decision is underdeveloped.

## 8. Package Risks

Call out risks explicitly:

- missing proof for a spoken claim
- reference pattern mismatch
- overuse of the same screen recording
- too many cuts on abstract lines
- B-roll stronger than the A-roll performance

Also call out fallback strategies:

- stay on face longer
- compress the beat
- swap to support proof
- drop a weaker claim

## Practical Rules

Use these defaults unless the piece clearly wants something else.

- Hook: keep the first face moment unless a proof image is stronger within the first second
- Overlay length: most short-form B-roll overlays should live in the `0.8s-2.5s` range
- Repetition: do not repeat the same distinctive B-roll too closely unless the recurrence is intentional
- Contrast videos: let the viewer feel both sides of the comparison, not just hear them
- Viewpoint videos: avoid over-proving every line and killing the voice

## Default Deliverables

Prefer a package with these sections:

1. edit thesis
2. video mode
3. beat map
4. B-roll proof inventory
5. edit decision timeline
6. missing assets and risks

That is enough for a real editor to start.
