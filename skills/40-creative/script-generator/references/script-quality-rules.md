# Script Quality Rules

Use these rules before returning any final script object.

## Core Principles

- start with a concrete viewer question, not a brand intro
- show or imply the problem or desire before ad feeling appears
- reveal the product only when it helps the hook
- keep one core message per asset
- make the mechanism legible, not just the outcome
- write for retention first, persuasion second
- hook before branding
- curiosity before explanation
- product after tension starts working
- mechanism before broad claims
- proof before CTA
- simplicity before cleverness

If the input violates these rules, fix structure first and wording second.

## Default Skeleton

The default target shape is:

1. Hook
2. Problem recognition or desire trigger
3. Proof or mechanism
4. Product role
5. Outcome
6. CTA

Compression is acceptable.

Missing `hook`, `proof or mechanism`, or `product role` is not acceptable for a final usable variant.

## Talking-Head / AI UGC Video Rules

Best when:

- human belief transfer matters
- explanation matters
- product handling or mechanism can be shown

Write it like speech:

- shorter clauses
- natural interruptions
- light self-correction
- selective lived-in expressions

Allowed example patterns:

- `I didn't expect this`
- `the weird part is`
- `what actually matters is`

Avoid:

- `revolutionary`
- `transforms your routine`
- generic adjective stacks
- brochure language

Use pacing slots instead of rigid duration doctrine:

- opening hook
- specific pain or desire
- product use or proof
- result or payoff
- CTA close

## Slideshow / Carousel Rules

Best when:

- the concept is visual
- proof can be broken into cards
- each frame can create micro-curiosity

Rules:

- one slide = one cognitive job
- keep each slide short
- make slide one a real hook
- ensure progression across cards
- do not flatten the whole message into feature bullets

## QA Rubric

Always check:

- `Viewer question clear?`
- `Ad feeling delayed enough?`
- `Product reveal timing ok?`
- `One core message only?`
- `Mechanism legible?`
- `Proof concrete enough?`
- `CTA soft enough?`
- `Video-native or slideshow-native?`
- `Main drift risk`
