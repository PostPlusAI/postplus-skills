# Pattern Families

Use these families to route a brief before writing prompts.

Choose one primary family.
Add one support family only when it clearly improves readability.

## Core Families

### 1. Aspirational Leisure / Soft Status

Use when the scene should sell a desirable state first.

Viewer question examples:

- "Where is this?"
- "How do I get that day?"
- "Can I copy this lifestyle setup?"

Opening mechanisms:

- payoff-first lifestyle frame
- relaxed selfie already inside the moment
- wide overhead scene with small humans inside a beautiful environment

Best for:

- hook
- lifestyle
- soft-sell CTA

Avoid:

- long setup before the good moment
- product hero shot in the first beat unless the product itself is the status fantasy

### 2. POV Leisure

Use when the viewer should feel physically placed in the scene.

Viewer question examples:

- "What is on that setup?"
- "What would it feel like to be there?"

Opening mechanisms:

- body plus environment in one frame
- first-person look down a surface, board, table, bed, counter, or trail

Best for:

- hook
- lifestyle
- inventory reveal

Avoid:

- abstract beauty shots with no embodied viewpoint

### 3. Comment-Request Reveal / Inventory Curiosity

Use when a list or kit reveal needs social permission.

Viewer question examples:

- "What exactly is in that setup?"
- "Can you show everything you brought?"

Opening mechanisms:

- real or implied comment request
- visible objects already in frame before the reveal starts

Best for:

- hook
- benefit
- creator

Avoid:

- flat product catalog opening
- full list too early

### 4. Problem-Solution Demo

Use when the product earns attention through friction removal.

Viewer question examples:

- "Does this actually fix the problem?"
- "Can this replace the harder way?"

Opening mechanisms:

- visible pain or cost contrast
- proof action already underway

Best for:

- benefit
- CTA
- testimonial

Avoid:

- spec list before the problem is legible

### 5. Proof Close-Up

Use when trust depends on visible mechanism or tactile evidence.

Viewer question examples:

- "Is that real?"
- "How is it working?"

Opening mechanisms:

- nozzle, texture, gauge, valve, hand interaction, screen, tool contact

Best for:

- benefit
- CTA
- testimonial

Avoid:

- beauty-commercial macro with no real use context

### 6. Creator Native Delivery

Use when the human presenter is the main trust asset.

Viewer question examples:

- "Why is this person recommending it?"
- "What happened to them?"

Opening mechanisms:

- mid-sentence admission
- direct camera confession
- interrupted routine or in-progress explanation

Best for:

- creator
- testimonial
- CTA

Avoid:

- over-scripted spokesperson energy

## Segment Mapping Defaults

Use these defaults unless the brief suggests a stronger route:

- `hook` -> `Aspirational Leisure`, `POV Leisure`, or `Comment-Request Reveal`
- `benefit` -> `Problem-Solution Demo` plus `Proof Close-Up`
- `cta` -> `Problem-Solution Demo` or `Creator Native Delivery`
- `creator` -> `Creator Native Delivery`, optionally supported by `Comment-Request Reveal`
- `lifestyle` -> `Aspirational Leisure` plus `POV Leisure`
- `testimonial` -> `Creator Native Delivery` plus `Proof Close-Up`

## Product Reveal Rule

By default:

- `hook`: product should not dominate the opening unless the hook is explicitly mechanism-led
- `benefit`: product should appear as proof, not as packaging
- `cta`: product can appear earlier, but still needs a reason inside the scene
- `creator`: product should feel carried by behavior, not by static showcase

## Output Shape

After routing, print:

```text

Support pattern:

```
